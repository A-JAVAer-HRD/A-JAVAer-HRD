4、Spring MVC入门

HTTP

  • HyperText Transfer Protocol 超文本传输协议

  • 用于传输HTML等内容的应用层协议

  • 规定了浏览器和服务器之间如何通信，以及通信时的数据格式。

https://developer.mozilla.org/zh-CN

HTTP 基础知识：

  • HTTP 协议是应用层协议，它的底层基于 TCP 连接
  • HTTP 协议规定了浏览器与服务器通信的四个步骤，依次是：建立连接、发送请求、接收响应、关闭连接
  • HTTP 请求报文中，包含请求方式、请求路径、协议版本、消息头等内容
  • HTTP 响应报文中，包含状态码、状态名、协议版本、消息头等内容

为什么访问一个页面会产生多个请求？

第一次访问到html,发现html引用了一些js，css再去请求。

Spring MVC

三层架构：

  • 表现层
  • 业务层
  • 数据访问层

MVC：是一种设计模式,是针对表现层设计的

  • Model：模型层

  • View：视图层

  • Controller：控制层

controllter 处理浏览器的请求，调用业务层处理数据，业务层调用数据层最后会对数据库进行操作，返回到controller层。controllter再把数据给model层封装，最后交给view渲染成html进行返回。

核心组件

  • 前端控制器：DispatcherServlet

  • front controller 前端控制器，就是DispatcherServlet。来处理浏览器的请求和响应。
  • DispatcherServlet 根据请求路径访问 controller，controller 把数据封装在 model 返回。DispatcherServlet 再把封装的 model 数据发送给 view 层，让 view 进行渲染，返回页面。

GET 请求也可以向服务器传数据，但是有两点：

  1. GET 请求会把参数用?拼接在路径上，参数直接显示，不安全。
  2. 参数拼接在路径上，路径也是由于长度的，参数过多，就会传不完。
