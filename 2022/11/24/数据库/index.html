数据库总结

一、绪论（Introduction）

1、数据库基本概念

数据库的四个基本概念：

  1. 数据 data
  2. 数据库 DataBase DB
  3. 数据库管理系统 DataBase Management System DBMS
  4. 数据库系统 DataBase System DBS

1.1、数据

  • 描述事物的符号 （数据的定义）
  • 数据库中存储的基本对象
  • 数字、文字、图形、声音、语言等多种表现形式
  • 数据符号 + 语义→→数据、

1.2、数据库

  • 存储在计算机内、有组织、可共享的数据集合
  • DB数据，按一定的数据模型组织、描述和存储
  • 数据冗余度小，数据独立性和扩展性高

1.3、DBS

DBS的组成部分：

  • 硬件：计算机（磁盘→ 数据库/DB/磁盘文件）
  • 软件：数据库管理系统（DBMS）、数据库应用系统（DBAS）
  • 个人：数据库管理员（DBA）、数据库用户

DBS的软件框架：

DBS逻辑结构：

三级模式、两级映射

三级模式：用户模式（外模式）、逻辑模式（模式）、内模式（存储模式）

两级映射与数据独立性：

  • 外模式/模式映射1：n

    模式改变，由DBA调整各外模式/模式映像， 可使外模式不变，支撑应用程序不修改， 保证数据与程序的逻辑独立性。

  • 模式/内模式映射1：1

    数据库的存储结构改变，由DBA调整模式/内模式映像，可使模式不变， 且无需修改应用程序，保证数据与程序的物理独立性。

系统结构：

  • Centralization(集中式)结构

    • DB物理上，在一个局域网中

  • Distribution(分布式)结构

    • DB物理上或逻辑上，分布在不同的网络中
      • Client/Server 结构
      • Browser/Server 结构
      • Parallel (并行)结构

实体(对象)+联系(关系)模型

数据模型分类：

• Network data model / 网状模型

• Hierarchical data model / 层次模型

• Relational data model / 关系模型

• Object-Relational data model /对象关系模型

数据模型Data Model

Network data model / 网状模型

Hierarchical data model / 层次模型

Relational data model / 关系模型

Object-Relational data model /对象关系模型

1.4、DBMS

数据管理软件，位于用户程序 与 操作系统之间 (专用) 。

功能：（四大主要方面）

  • 数据定义—建库、建表、创建用户等。用DDL表达
  • 数据操纵—插入数据、修改和删除数据等。用DML表达
  • DB运行管理—数据使用者及其使用权限管理等。用DCL表达
  • DB建立和维护—数据库备份、回复等

DBMS的功能实施，用SQL(Structure Query Language)语言发指令 。

➢ Data Define Language（DDL）

➢ Data Management Language（DML）

➢ Data Control Language（DCL）

1.5、总结概念和属性

DBMS一次可以管理多个数据库。
▪ 在大型DBMS中，一个DB存储在一个DISK文件中。
▪ 许多表与结合在一个数据库中。
▪ 在RDB中，一个属性必须是非结构化数据。
▪ 磁盘文件。
▪ 数据库。
▪ 表、实体、关系
▪ 行、记录、元组
▪ 列、字段、属性

对于一个表：

  • 整个表称为表、实体、关系。
  • 表中的每一行行称为行、记录、元组。
  • 表中的每一列称为列、字段、属性。

在关系模型中，表的列必须包含

  • 单个值
  • 非结构化值
